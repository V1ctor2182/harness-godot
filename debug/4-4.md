### 4. 数据同步 Design Decision
- 讨论了 disk 为主 vs DB 为主
- 最终选择 **DB 为主 + Disk 镜像**：写入只走 DB，syncToDisk() 是 post-hook
- 内容变更实时 sync，qualityScore batch sync（每 cycle 结束）
- Disk 永远 = DB 快照，可 git diff / PR review

### 5. 双 repo rooms 设计
- Harness rooms (26) = "怎么工作"（PR 格式、测试超时、container 规则）
- Game rooms (~32) = "做什么"（种植公式、基因数量、战斗 damage）
- 同一个 MongoDB，`repo` 字段区分
- Context Builder 合并两边 specs 注入 agent

### 6. Flag + Reflect 机制设计
- 实时 flag：Coder/Tester/Reviewer 在 structured output 加 `flags[]`
- 6 种 flag types：scope_concern, missing_dependency, prd_issue, new_system_needed, milestone_complete, tech_debt
- Blocker flag → 跳过自动 retry（结构性问题重试没用）
- Rate limited → 也不自动 retry（浪费钱）
- 每 3 个 cycle Orchestrator 做 reflect：读 flags + metrics + Room 变化，输出 milestone_health + proposed_changes
- Reflect 结果需 human 确认才生效

### 7. Orchestrator Plan Review (GitHub Issue #29)
- Orchestrator 输出 `questions[]` alongside plan
- Plan 进入 `pending-review`，Dashboard 展示问题 + 回答 UI
- Human 回答后才 dispatch tasks
- Q&A 沉淀为 decision specs 写入 Feature Rooms
