## P3. 并发模型

```
  ┌─────────────────────────────────────────────────────────────────────┐
  │                    CONCURRENCY MODEL                                │
  │                    单进程 Node.js, 单 Poll Loop                     │
  └─────────────────────────────────────────────────────────────────────┘

  核心约束:
  ┌──────────────────────────────────────────────────────────────────┐
  │ ✓ 单进程 — 一个 Express server，一个 poll loop                  │
  │ ✓ 单实例 — Docker Compose 部署，不支持水平扩展                   │
  │ ✗ 无分布式锁 — 多实例会导致 job double-claim                    │
  │ ✗ 无 message queue — 纯 DB polling，非 Redis/RabbitMQ           │
  └──────────────────────────────────────────────────────────────────┘

  ═══════════════════════════════════════════════════════════════════════
  POOL SLOTS — 并发槽位
  ═══════════════════════════════════════════════════════════════════════

  ┌────────────────────────────┐    ┌────────────────────────────────┐
  │      AGENT POOL            │    │       INFRA POOL               │
  │      默认 2 槽位            │    │       默认 3 槽位               │
  │                            │    │                                │
  │  spawn (coder/tester/      │    │  wait-for-ci                   │
  │    reviewer/orchestrator/  │    │  apply-plan                    │
  │    integrator/curator)     │    │  advance-cycle                 │
  │  spawn-tester              │    │  next-cycle                    │
  │  curate-specs              │    │  reload                        │
  │  plan-review               │    │  cleanup-prs                   │
  │                            │    │  run-gut-tests                 │
  │  ← Docker 容器 (重量级)     │    │  run-integration-tests         │
  │  ← 每个 4GB + 1 CPU        │    │  run-visual-tests              │
  │                            │    │  run-prd-compliance            │
  │                            │    │  create-fix-task               │
  │                            │    │  validate-assets               │
  │                            │    │                                │
  │                            │    │  ← 轻量级 (无 Docker 容器)      │
  │                            │    │  ← DB 操作 / GitHub API 调用    │
  └────────────────────────────┘    └────────────────────────────────┘

  ═══════════════════════════════════════════════════════════════════════
  POLL CYCLE — 每 5 秒一次
  ═══════════════════════════════════════════════════════════════════════

  ┌──────────────────────────────────────────────────────────────────┐
  │                                                                  │
  │  setInterval(pollOnce, 5000)                                     │
  │                                                                  │
  │  pollOnce():                                                     │
  │  ┌────────────────────────────────────────────────────────────┐  │
  │  │                                                            │  │
  │  │  ① Check processing flag                                   │  │
  │  │     processing === true? ──► return (skip this tick)       │  │
  │  │     processing = true                                      │  │
  │  │                     │                                      │  │
  │  │                     ▼                                      │  │
  │  │  ② Check system mode                                      │  │
  │  │     control.mode === 'killed'? ──► return                  │  │
  │  │     control.mode === 'paused'? ──► return                  │  │
  │  │                     │                                      │  │
  │  │                     ▼                                      │  │
  │  │  ③ Count active jobs per pool                              │  │
  │  │     agentActive = count({ pool: 'agent', status: 'active'})│  │
  │  │     infraActive = count({ pool: 'infra', status: 'active'})│  │
  │  │                     │                                      │  │
  │  │                     ▼                                      │  │
  │  │  ④ Compute available slots                                 │  │
  │  │     agentSlots = AGENT_POOL_SIZE - agentActive             │  │
  │  │     infraSlots = INFRA_POOL_SIZE - infraActive             │  │
  │  │                     │                                      │  │
  │  │                     ▼                                      │  │
  │  │  ⑤ For each available slot:                                │  │
  │  │     findOneAndUpdate({                                     │  │
  │  │       status: 'pending',                                   │  │
  │  │       pool: poolType                                       │  │
  │  │     }, {                                                   │  │
  │  │       $set: { status: 'active', claimedAt: new Date() }    │  │
  │  │     })                                                     │  │
  │  │     ← MongoDB 原子操作，保证不会 double-claim              │  │
  │  │                     │                                      │  │
  │  │                     ▼                                      │  │
  │  │  ⑥ Check approval gate                                    │  │
  │  │     job.requiresApproval && !job.approved?                 │  │
  │  │       YES ──► hold (status stays pending)                  │  │
  │  │       NO  ──► check autoApprovalCategories                 │  │
  │  │                     │                                      │  │
  │  │                     ▼                                      │  │
  │  │  ⑦ Dispatch handler                                       │  │
  │  │     handlers[job.type](job) ──► async, non-blocking        │  │
  │  │                                                            │  │
  │  └────────────────────────────────────────────────────────────┘  │
  │                                                                  │
  │  finally: processing = false                                     │
  │                                                                  │
  └──────────────────────────────────────────────────────────────────┘

  ═══════════════════════════════════════════════════════════════════════
  JOB CLAIMING — 原子性保证
  ═══════════════════════════════════════════════════════════════════════

  ┌──────────────────────────────────────────────────────────────────┐
  │                                                                  │
  │  findOneAndUpdate 是 MongoDB 原子操作:                           │
  │                                                                  │
  │  Filter:  { status: 'pending', pool: 'agent' }                  │
  │  Update:  { $set: { status: 'active', claimedAt: now } }        │
  │  Sort:    { priority: -1, createdAt: 1 }  ← 高优先级 + 先进先出 │
  │  Return:  修改后的文档 (returnDocument: 'after')                 │
  │                                                                  │
  │  并发安全:                                                       │
  │  ┌────────────────────────────────────────────────────────────┐  │
  │  │ 即使有两个 poll 重叠执行（理论上不会，有 processing 守卫） │  │
  │  │ findOneAndUpdate 保证同一个 job 只被一个 caller 拿到      │  │
  │  │ 第二个 caller 的 filter 匹配不到已变 active 的 job        │  │
  │  └────────────────────────────────────────────────────────────┘  │
  │                                                                  │
  └──────────────────────────────────────────────────────────────────┘
```
